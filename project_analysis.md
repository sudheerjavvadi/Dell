# Project Analysis: Dell Network Analyzer

This repository contains a full-stack, multi-component cyber-security tool designed to analyze network traffic logs (specifically formatted like the UNSW-NB15 dataset) and detect anomalous or malicious activities, such as port scanning, backdoor connections, and coordinated attacks.

The project is structured into three main sub-directories:

## 1. Heuristics-Based Analyzer (`/python_analyzer`)
This component uses traditional, rule-based and statistical heuristics to identify suspicious network flows. It relies heavily on `pandas` for processing large CSV files efficiently.

**Key Files & Logic:**
*   **`analyzer.py`**: Contains the core `PortScanAnalyzer` class. It implements a sliding window algorithm to monitor connection histories for source IPs. It detects:
    *   **Vertical Port Scans:** Single IP scanning many ports on a single host.
    *   **Horizontal Port Scans:** Single IP scanning the same port across many different destination hosts.
    *   **Strobe Scans:** A combination of scanning multiple hosts and multiple ports.
    *   **Backdoor/C2 Connections:** Traffic sent to known C2 ports (e.g., 4444, 8080) with low overall volume.
    *   **Stealth Scans:** High ratio of failed/interrupted connections.
*   **`advanced_analyzer.py`**: Contains `AdvancedPortScanAnalyzer`, which extends the base analyzer to identify more sophisticated attack patterns:
    *   **Distributed Campaigns:** Correlates multiple source IPs targeting the same destination ports in a tight time window.
    *   **Slow Port Scans:** Attackers scanning very slowly over an extended period to evade rate-based detection.
    *   **Common Ports Probes:** Targeting a specific subset of widely-exploited ports across the network.
*   **Data Handling:** The system supports parsing both headered and headerless network logs and generates output CSVs/reports (e.g., `suspicious_ips_report.csv`).

## 2. Machine Learning Analyzer (`/ml_analyzer`)
This component provides a secondary layer of detection using Deep Learning techniques, utilizing `scikit-learn` to train a model on labeled network data.

**Key Files & Logic:**
*   **`train_and_report.py`**: The main execution script.
    *   **Training:** It loads the `UNSW_NB15_training-set.csv`, scales the numeric features using `StandardScaler`, and trains a Multi-Layer Perceptron (MLP) Neural Network classifier.
    *   **Inference:** It runs the trained model against a sample of another dataset (`UNSW-NB15_2.csv`) to predict the probability of malicious activity per network flow.
    *   **Aggregation:** It groups the results by Source IP, calculating aggregate statistics (total bytes, duration, peak rates) and assigns a severity rating (Critical, High, Medium, Normal) based on the model's confidence scores.
    *   **Export:** It synthesizes these findings into a JSON report (`ml_report_data.json`) and **directly saves it into the `react_dashboard/public` folder** so the frontend can display it immediately.

## 3. Frontend Visualization (`/react_dashboard`)
A modern web application built to visualize the output generated by the Python and ML analyzers.

**Key Technologies & Structure:**
*   **Frameworks:** Built with React 19, Vite, and vanilla CSS for styling (`App.css`, `index.css`).
*   **Dependencies:** Uses `Chart.js` (via `react-chartjs-2`) for rich data visualization and `lucide-react` for iconography.
*   **Architecture:**
    *   `src/pages` & `src/App.jsx`: Houses the UI structure. The dashboard likely fetches the generated JSON reports (like `ml_report_data.json`) to populate statistical cards, severity distribution charts, and tables of suspicious IPs.
    *   `src/workers`: Suggests the use of Web Workers to parse and handle large JSON payload arrays in the background to prevent UI blocking.

## Workflow Summary
1. **Data Ingestion:** Network logs (CSV) are placed in the `archive` directory.
2. **Analysis Execution:** The user runs either the heuristic analyzer or the `train_and_report.py` ML script.
3. **Data Bridging:** The Python scripts crunch the data, assign severities/classifications, and export JSON data directly into the React app's `public` folder.
4. **Visualization:** The React dashboard is served locally (`npm run dev`) to provide a graphical interface for security analysts to review the threats, graphs, and connection statistics.
